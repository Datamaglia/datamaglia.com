Get Started wth Datamaglia Recommendations

This tutorial will show how Datamaglia Recommendations can be used to enhance user engagement in a photo browsing app. We will show how to request a Datamaglia API key, how to configure your Datamaglia application and how to use the Datamaglia Recommendations API to generate personalized recommendations and show these to the user.

Introducing Picmaglia

This tutorial will assume we are a developer with a basic photo browsing app and how we can add personalized recommendations.

Overview

These are the basic features that we've already implemented for Picmaglia. Users are able to

  • Browse photos: view the most recently shared photos.
  • Favorite photos: select photos they like and save these for viewing later.
  • View favorited photos: see all the photos that user has favorited.
  • View recommended photos What photos is this user likely to be interested in?

These features are great, but we'd like to increase user engagement with our app by adding a way for users to discover new photos (other than by viewing recent photos). We'd like to be able to suggest new photos to users based on the photos they have previously favorited.

Configure app in Datamaglia Console

The first thing we'll need to do is create a Datamaglia API token for our app. We do this in the Datamaglia Managment Console. You'll need an invite code to create an account on the console. If you haven't signed up yet, you can sign up here for early access to the Datamaglia platform.

Create an application

After logging into the management console we must first create an application.

Each developer can create multiple applications. Here we will create an application called Picmaglia to represent our photobrowsing app.

Create a subgraph

To make use of Datamaglia Recommendations each application must have at least one subgraph. As explained in the How it works section, Datamaglia models your application data as a graph. This allows us to build a powerful flexible representation of user preferences from which we can generate personalized recommendations. A subgraph defines the datatypes for a particlar (Source -[RELATIONSHIP]-> Target) combination.

The Source node of the subgraph is the data type of the object initating an action toward another object. With Picmaglia we are interested in using data about what photos a user has likes, therefore User is the source type for our subgraph. We will use a relationship type of LIKES to represent the action of favoriting. Finally, the Target type is Photo since that is the object the user is favoriting.

It is important to note here that recommendations are generated for the target node. This is always the case, since we are effectively predicting edges in the graph to generate recommendations.

Create an API token

Now we need to create an API key to allow us to make authenticated requests against the Datamaglia API. We can choose which permissions to grant the API token. It is important to note that each request made to the Datamaglia API must contain this API Token in the request header. View the API documentation for more information.

Insert data

Now that we've configured our application, subgraph and API token we can start using the Datamaglia API.

In order to generate recommendations for our users we first need to tell the Datamaglia Recommendations API about our user preferences. We do this by posting data describing the (User)-[LIKES]->(Photo) subgraph:


// POST /applications/{app}/subgraphs/{subgraph}/data/relationships/
// BODY:
{
    "entities": [
        {
            "type": "Likes",
            "weight": 1.0,
            "source": {
                "id": {
                    "email": "johndoe@example.com"
                }
            },
            "target": {
                "id": {
                    "imdb": "tt0398286"
                }
            }
        },
        {
            "type": "Likes",
            "source": {
                "id": {
                    "email": "johndoe@example.com"
                }
            },
            "target": {
                "id": {
                    "imdb": "tt1217209"
                }
            }
        }
    ]
}
